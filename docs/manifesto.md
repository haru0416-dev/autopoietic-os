# Manifesto

Autopoietic OS is not an application launcher with an AI assistant. It is a research environment for growing the operating system itself.

The operating system is treated as a living configuration body:

- the flake and modules are the genome;
- systemd units, CLIs, packages, and devShells are organs;
- generations are lineage, not merely recovery points;
- failed builds are evolutionary evidence, not trash;
- memory is an operational history that must be read before future mutations.

The central rule is:

> AI should not primarily operate Linux by issuing commands. AI should mutate Nix configuration, submit that mutation to verifiers, and let accepted generations become the next body.

The first success condition is self-understanding, not autonomy. A system that cannot describe itself as changeable structure cannot safely mutate itself.
